React Native打包实战：配置 EAS Build 生成 APK

首页frontend

在 React Native (Expo) 开发完成后，如何将代码变成可以在真机上安装的 .apk 文件？本文记录了使用 EAS Build (Expo Application Services) 进行云端打包的详细步骤。

# 1. 准备工作

首先，我们需要安装 Expo 提供的构建工具 eas-cli 并登录账号。若没有账号则需要注册 https://expo.dev/

# 全局安装 EAS CLI
npm install -g eas-cli

# 登录 Expo 账号
eas login

# 2. 项目标识配置 (app.json)

在打包前，必须确定 App 的名称和唯一的包名（Package Name）。打开项目根目录下的 app.json 进行修改：


JSON
{
  "expo": {
    "name": "中医智能面诊",        // App 在手机桌面上显示的名字
    "slug": "tcm-face-diagnosis", // Expo 项目标识（英文）
    "version": "1.0.0",
    "android": {
      "package": "com.koen.tcmface", // 【关键】安卓包名，必须唯一
      "adaptiveIcon": {
        "foregroundImage": "./assets/adaptive-icon.png",
        "backgroundColor": "#ffffff"
      }
    },
    "ios": {
      "bundleIdentifier": "com.koen.tcmface" // iOS 包名
    }
  }
}

# 3. 构建配置 (eas.json)

默认情况下，EAS Build 会生成 .aab 文件（用于上架 Google Play）。如果我们想直接生成可安装的 .apk 文件，需要修改 eas.json。

运行初始化命令：

eas build:configure

然后修改生成的 eas.json，重点是添加 preview 字段中的 buildType: "apk"：

JSON
{
  "cli": {
    "version": ">= 7.0.0"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "android": {
        "buildType": "apk" 
      }
    },
    "production": {
      "android": {
        "buildType": "app-bundle"
      }
    }
  }
}

# 4. 执行打包

一切准备就绪后，运行以下命令开始云端构建：

eas build -p android --profile preview


-p android: #指定打包安卓平台。

--profile preview: #使用我们刚才配置的 preview 模式（生成 APK）。

构建完成后，终端会生成一个二维码和下载链接，直接下载安装即可。
