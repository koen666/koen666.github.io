# 1️.screen 是什么？

  • screen 是一个终端复用器（terminal multiplexer）。

  • 它可以在一个终端窗口中运行多个“虚拟终端”，允许你在后台保持会话。

  • 常见用途：

    • 在远程服务器上运行长时间任务，即使断开 SSH 也能继续执行。
    • 在一个终端里同时打开多个 shell。
    • 在同一会话中共享终端（多人操作同一个 session）。

# 2️.安装

大多数 Linux 发行版默认有：

screen --version

如果没有安装：

# Ubuntu / Debian
sudo apt install screen

# CentOS / RHEL
sudo yum install screen

# 3️.基本命令

# (1) 启动一个新的 screen 会话

screen -S mysession
  • -S mysession：给会话取名字，方便管理。
  • 执行后，你会进入一个新的终端环境，和普通 shell 一样。

# (2) 在 screen 中操作

  • 新建窗口: Ctrl+a 然后 c
  • 切换窗口: Ctrl+a 然后 n（下一个）、p（上一个）
  • 查看窗口列表: Ctrl+a 然后 "

# (3) 暂时离开（detach）会话

Ctrl+a d
  • 会让 screen 会话在后台继续运行，终端回到原来的 shell。

# (4) 恢复（attach）会话

screen -r mysession
  • 恢复之前 detach 的会话。
  • 如果有多个会话：
screen -ls
# 列出所有会话
screen -r 会话ID

# (5) 杀掉会话

exit
  • 在 screen 内直接 exit 会结束该窗口和会话。
    或者：
screen -X -S mysession quit

# 4️.高级技巧

  1. 后台运行任务
screen -S mytask -dm bash -c "python myscript.py"
  • -dm 表示启动会话并立即后台运行。
  • 可以 screen -r mytask 查看进度。
  1. 日志记录
Ctrl+a H
  • 会将当前窗口的输出记录到 screenlog.0 文件。
  1. 共享会话
screen -x mysession
  • 可以和其他用户同时操作同一个 session（需要权限）。

# 5️.总结

screen 的核心价值是：

  • 断开重连：网络断了也不会影响任务。
  • 多窗口管理：在一个终端里搞定多个任务。
  • 后台执行：任务可持续运行，不占本地终端。
