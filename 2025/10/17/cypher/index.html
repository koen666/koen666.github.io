# 一、Cypher是什么

Cypher是操作图数据库（Neo4j） 的语言。
图数据库里的数据不是表格（像MySQL的表），而是像“地图”一样的结构：

  • 有很多“点”（比如“人”“病”“药”），这些“点”叫节点（Node）。
  • 点和点之间有“线”连接（比如“人患病”“药治人”），这些“线”叫关系（Relationship）。

Cypher的作用就是：描述“点和线”的样子，让数据库找到你要的数据。

# 二、最基础：节点（Node）怎么表示？

节点是图里的“实体”（比如“张三”“感冒”），写法是：
(变量名:标签 {属性})

# 拆解每个部分：

  • ()：必须用圆括号把节点包起来，相当于“这是一个节点”。
  • 变量名：给节点起个临时小名（比如用p代表“人”，d代表“病”），方便后面引用。
  • :标签：节点的“类别”（比如:Person表示“人”，:Disease表示“疾病”），一个节点可以有多个标签（比如:Person:Patient表示“人且是患者”）。
  • {属性}：描述节点的特征，是键值对（比如name: "张三"，age: 30），用大括号包起来。

# 例子：

// 一个“人”节点：变量p，标签Person，属性name和age
(p:Person {name: "张三", age: 30})

// 一个“疾病”节点：变量d，标签Disease，属性name
(d:Disease {name: "感冒"})

你可以理解为：
(p:Person {name: "张三"}) → “有个叫p的东西，它是‘人’，名字叫张三”。

# 三、关键：关系（Relationship）怎么表示？

关系是节点之间的“连接”（比如“张三患感冒”），写法是：
-[变量名:类型 {属性}]->

# 拆解每个部分：

  • -[ ]-：中括号代表“这是一个关系”，横线连接两个节点。
  • ->：箭头表示方向（比如A->B表示“A到B的关系”，无方向可以写成--）。
  • 变量名：给关系起个临时小名（比如r），方便引用。
  • :类型：关系的“类别”（比如:患表示“患病”，:治疗表示“可以治疗”），每个关系只能有一个类型。
  • {属性}：描述关系的特征（比如:患的关系可以有时间: "2023-10-01"）。

# 例子：

// “张三患感冒”的关系：p（张三）通过r（患）指向d（感冒）
(p:Person {name: "张三"})-[r:患 {时间: "2023-10-01"}]->(d:Disease {name: "感冒"})

可以理解为：
(p)-[r:患]->(d) → “p和d之间有个叫r的关系，类型是‘患’，意思是p患了d”。

# 四、核心操作：查数据（MATCH + RETURN）

最常用的操作是“找数据”，用MATCH描述你要找的“点和线的模式”，用RETURN返回结果。

# 1. 只查节点

比如：“找所有叫‘张三’的人”

// 步骤1：用MATCH描述模式（找标签是Person，name是张三的节点，变量叫p）
MATCH (p:Person {name: "张三"})
// 步骤2：用RETURN返回结果（返回p的所有信息）
RETURN p

再比如：“找所有年龄大于30的人，只返回他们的名字”

MATCH (p:Person)  // 找所有标签是Person的节点
WHERE p.age > 30  // 加条件：年龄大于30（WHERE是过滤条件）
RETURN p.name     // 只返回名字

# 2. 查关系（节点+关系）

比如：“找‘感冒’这种病有哪些伴随的症状”

// 模式：疾病d（名字是感冒）通过关系r（类型是伴随症状）指向症状s
MATCH (d:Disease {name: "感冒"})-[r:伴随症状]->(s:Symptom)
// 返回疾病名、关系的程度（比如“常见”）、症状名
RETURN d.name, r.degree, s.name

如果数据库里有“感冒伴随发烧、咳嗽”，结果会是：

d.name r.degree s.name
感冒 常见 发烧
感冒 较常见 咳嗽

# 3. 查路径（多个关系连起来）

比如：“找张三患的病有哪些症状”（路径：人→患→病→伴随→症状）

// 模式：p（张三）-患→d（病）-伴随→s（症状）
MATCH (p:Person {name: "张三"})-[r1:患]->(d:Disease)-[r2:伴随]->(s:Symptom)
RETURN p.name, d.name, s.name  // 返回张三、病名、症状名

# 五、其他常用操作：增删改

# 1. 创建（CREATE）

比如：“新增一个叫‘李四’的人，年龄25”

CREATE (p:Person {name: "李四", age: 25})

比如：“给‘李四’和‘感冒’加一个‘患’的关系，时间是今天”

// 步骤1：先找到李四和感冒这两个节点
MATCH (p:Person {name: "李四"}), (d:Disease {name: "感冒"})
// 步骤2：创建他们之间的关系
CREATE (p)-[r:患 {时间: "2023-10-02"}]->(d)

# 2. 修改（SET）

比如：“把张三的年龄改成31，再加上性别‘男’”

// 先找到张三
MATCH (p:Person {name: "张三"})
// 用SET修改/新增属性
SET p.age = 31, p.gender = "男"

# 3. 删除（DELETE）

删除节点前，必须先删除它的所有关系（不然“线”断了会报错）。
比如：“删除‘发烧’这个症状节点”

// DETACH DELETE会自动删除该节点的所有关系，再删节点
MATCH (s:Symptom {name: "发烧"})
DETACH DELETE s
